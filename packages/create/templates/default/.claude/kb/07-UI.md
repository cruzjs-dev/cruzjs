# UI Patterns

## Stack

- **React** with React Router v7
- **Tailwind CSS** for layout and styling
- **`@cruzjs/ui`** for components (Button, Modal, Input, Badge, Tabs, Select, Card, Alert, toasts, and ~110 more)

Author pages with Tailwind utility classes + `@cruzjs/ui` components. Do **not** import components directly from `@chakra-ui/react`.

## Vite SSR Config (leave it alone)

`@chakra-ui/*`, `@emotion/*`, and `framer-motion` are listed in `ssr.noExternal` in `vite.config.ts`. These are transitive framework dependencies — **do not remove them**. If Chakra-derived styles render broken on first load (e.g. `border-radius:2xl` in the HTML source instead of a CSS variable), the `ssr.noExternal` config has been broken. Keep it as generated. You still author your own pages with `@cruzjs/ui` + Tailwind, not raw Chakra.

## Route Files

Route files live inside features and use `export default`:

```typescript
// src/features/notes/routes/index.tsx
import { trpc } from '@/trpc/client';

export default function NotesPage() {
  const { data, isLoading } = trpc.notes.list.useQuery();

  if (isLoading) {
    return <div className="p-8">Loading...</div>;
  }

  return (
    <div className="max-w-2xl mx-auto p-8">
      <h1 className="text-2xl font-bold mb-6">My Notes</h1>
      {data?.map((note) => (
        <div key={note.id} className="p-4 border rounded mb-4">
          <h3 className="font-semibold">{note.title}</h3>
        </div>
      ))}
    </div>
  );
}
```

Register in `src/routes.ts` (via `createCruzRoutes`):

```typescript
import { type RouteConfig, route, index, layout, prefix } from '@react-router/dev/routes';
import { createCruzRoutes } from '@cruzjs/core/routing';
import { registerCruzStartRoutes } from '@cruzjs/start/routing';

export default createCruzRoutes({
  route, index, layout, prefix,
  dir: import.meta.dirname,
  framework: { registrars: [registerCruzStartRoutes] },
  routes: [
    index('routes/index.tsx'),
    ...prefix('notes', [
      index('features/notes/routes/index.tsx'),
      route(':id', 'features/notes/routes/$id.tsx'),
    ]),
  ],
}) satisfies RouteConfig;
```

## Toasts

```typescript
import { useToast } from '@cruzjs/ui';

const toast = useToast();
toast({ title: 'Created', status: 'success' });
toast({ title: 'Something went wrong', description: err.message, status: 'error' });
// status: success | error | info | warning
```

`ToastProvider` is mounted at the app root (already wired in `root.tsx`) — just call `useToast()` in your components.

## tRPC Hooks

```typescript
// Query
const { data, isLoading, error, refetch } = trpc.product.list.useQuery();

// Query with input
const { data } = trpc.product.get.useQuery({ id: productId });

// Conditional
const { data } = trpc.product.get.useQuery({ id }, { enabled: !!id });

// Mutation
const createMutation = trpc.product.create.useMutation({
  onSuccess: () => { refetch(); toast({ title: 'Created', status: 'success' }); },
  onError: (err) => { toast({ title: err.message, status: 'error' }); },
});
```

## Permission-Based UI

```typescript
const canManage = currentUserRole === 'OWNER' || currentUserRole === 'ADMIN';
const canDelete = currentUserRole === 'OWNER';

return (
  <div>
    {canManage && <Button onClick={handleCreate}>Create</Button>}
    {canDelete && <Button variant="danger" onClick={handleDelete}>Delete</Button>}
  </div>
);
```
